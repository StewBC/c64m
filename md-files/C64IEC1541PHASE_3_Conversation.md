**1. ROM intercept addresses — blocker**

// Standard 1541 ROM (DOS 2.6: 325302-01 + 901229-06AA combined).
// Addresses verified from dos.lbl generated by assembling github.com/mist64/1541rom
// (dos1541 source, 325302-01 / 901229-06 variant).
//
// Intercept strategy: patch at the job-queue level.
// The ROM writes a READ job ($80) into jobs[N] (zero page $00-$05),
// sets hdrs[N*2] = track and hdrs[N*2+1] = sector (zero page $06-$11),
// and waits in watjob()/tstjob() for jobs[N] to go < $80 (job done).
// The controller IRQ (dskint at $EAA0) normally picks up the job and
// calls reed ($F4CA) to do the physical read.
//
// We intercept at reed ($F4CA): substitute D64 sector data into the
// correct buffer and jump the 1541 CPU's PC to read40 ($F505, the
// success return path), which stores $01 in jobs[jobn] via errr ($F969).
//
// RAM locations (zero page unless noted):
#define C1541_ZP_JOBS           0x0000  // jobs[0..5]: job queue (6 bytes)
#define C1541_ZP_HDRS           0x0006  // hdrs[0..11]: track/sector pairs (12 bytes)
#define C1541_ZP_JOBN           0x003F  // jobn: index of current job
#define C1541_ZP_TRACK          0x0080  // track: current track (DOS layer)
#define C1541_ZP_SECTOR         0x0081  // sector: current sector (DOS layer)

#define C1541_ROM_REED          0xF4CA  // entry: physical sector read (intercept here)
#define C1541_ROM_READ40        0xF505  // success return: sector in buffer, sets job ok
#define C1541_ROM_ERRR          0xF969  // job completion: stores result in jobs[jobn]

#define C1541_RAM_SECTOR_BUF    0x0300  // buff0: first sector buffer (job 0)
                                        // buff1 = $0400 (job 1), buff2 = $0500 (job 2)
                                        // buffer for job N = $0300 + N*$0100

#define C1541_JOB_READ          0x80    // job code: read
#define C1541_JOB_OK            0x01    // result: success
#define C1541_JOB_ERROR         0x02    // result: error (generic)
The intercept logic in c1541_satisfy_sector_read() should:

1. Read jobn from $3F to get the active job index N
2. Read track from hdrs[N*2] ($06 + N*2) and sector from hdrs[N*2+1]
3. Fill $0300 + N*$100 with the 256 bytes from the D64
4. Set the 1541 CPU's PC to $F505 (read40) — the ROM then runs to errr and marks the job done normally
This is cleaner than jumping directly to errr because read40 sets up some state (the block ID check at read28 is skipped, which is fine since we're providing clean data) and then falls into errr naturally, letting the ROM's own job-completion path run.

Answer:
A. Register state at read40
Clean — no register dependencies. read40 is simply:

read40  lda #1      ; load A with success code
        jmp errr    ; errr does: ldy jobn / sta jobs,y / ...
It immediately loads A with #1 itself, discarding whatever was in A. Y and X are also overwritten by errr immediately (ldy jobn, then ldx savsp). You can jump to $F505 with any register state and it works correctly.
B. Valid range for jobn
numjob = 6 and bfcnt = 5. These are deliberately different. The bufind table confirms it:

bufind  .byte $03,$04,$05,$06,$07,$07
Jobs 0–4 map to buffers at $0300–$0700 (pages 3–7, all within the 2KB RAM at $0000–$07FF). Job 5 maps to page $07 — same as job 4, a deliberate alias used for the command/error channel overhead. Critically, job 5 is never a READ job — it's reserved for internal DOS use (command and error channels). The controller only dispatches READ jobs to jobs 0–4.
So in practice jobn during a READ will always be 0–4, and the buffer formula $0300 + N*$0100 is safe for all of them. That said, a defensive if (N >= 5) return error guard costs nothing and is worth adding — it protects against any unexpected ROM state and makes the code's assumptions explicit.
